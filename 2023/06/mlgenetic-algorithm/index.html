[ML]Genetic Algorithm

June 2, 2023 in ML

The genetic algorithm(GA), a random adapative global search algorithm gains insight from the natural process of gene inheritance.

In the GA, each feasible solution is a chromosome consisting of a code string of which the unit is called gene. The spirit of GA is to contrast different chromosome according to the evaluation function and have a tendency to retain the more adaptive ones and have others mutate via population crossover.

It is worth noting that we cannot ensure GA to get hands on the global minima. Despite the drawback, it is still possible to find local minima well enough.

To get closer to GA, standing there are problems below urging to solve.

  1. How to code the chromosome and initialize the gene group?
  2. How to design the fitness-assessment function to pick out the most outstanding ones?
  3. How to define the rule of population crossover and mutation?

Later on, we compare the fitness value of new-born with that of the previous generation. If the new one achieves better perfomance, then it will take the place, or otherwise be cast away.

1. To Encode the Chromosome.

The specific form of chromosome is up to the problem to solve.

For example, in the 0-1 knapsack problem with 4 problems in total, we mark one object 1 if taken and 0 on the contrary. Therefore, Binary Encoding fits such scenario as Combinaton Optimization Problems in discrete field. and to solve problems to do with something like rotation where a slight variance can make a great difference, it is smarter to turn to Grey Encoding, a opitmized binary encoding which changes one gene at a time.(For instance, the mutation from 0000 to 0001 is valid while 0000 shifting to 1001 is not)

As to look for the global minima of a continous function, more commonly, we opt for Floating-Point Encoding where a chromosome is simply a real value or vector; and to deal with problems like classification(where a chorosome stands for a label and the gene unit implies the relationship between different catagories) and text-generation, Alphabet Encoding comes into sight by nature.

Generally speaing, floating-point encoding outweights binary encoding where a high accuracy is required and a larger space waits to search etc.

In most cases, we create the chromosome population following the encoding rules and the problem restrictions above, but the value of their unit is mostly randomized. But if we can shrink the feasible range or to know the approximate range of the best solution, it is of a higher possiblity to get the answer.

2. To Pick Out the Competitive Chromosomes

Supposing that we have already figured out the fitness value of each chromosome, say 1,2,3,4 for example, and apply the widely-used selection opeators below to pick out N candidates for the new group which means you need to execute N selections:

  • Roulette Wheel Selection. is a resampling with replacement method. The possibility of each chorosome is worked out by the fitness value account in all (1/10,2/10,3/10,4/10 respectively). Then generate a random value to pick out one according to the possibility distribution above. (Such method is a similar to the Roulette Wheel where the higher fitness value is, the larger the zone is)

  • Stochastic Tournament It is Based on the distribution from Roulette Wheel Selection but this time we pick out two with the distribution and choose the higher one.

  • Elitism. Pick out the best one directly and apply Roulette Wheel Selection to get the rest N-1 chorosome.

  • Expected Value Selection Still we get the possibility distribution and work out the expected value of survivors. For instance, (N=4), 4×1/10=0.4,4×2/10=0.4,4×3/10=0.9, 4×4/10=1.6. If an individual is picked out to crossover, then the amount -0.5; or otherwise -1. If the survivor amount becomes negative, then it has lost the chances to be seleceted.

  • Deterministic Selection Calculate the expected survivor amount and sort them up. Say, 1.6,0.9,0.4,0.4, then we take 1,0,0,0 into the next generation. But that is not enough, then we range them up according to the decimal part(fraction part), say 0.9,0.6,0.4,0.4 and take the first three away.

  • Best Preservation Strategy The best won’t engage in the crossover or mutation process but instead replace the worst generated gene.

3. Crossover ahnd Mutation

A pair of genes crossovers with the possibility p. And way of crossover differs as well.

  • One-point Crossover Randomly choose a crossover point, say the random number is 2, then swap the second part of gene 100 and gene 111, thus getting 110,101. Likewise, we have Two-Point Crossover and Multipoint Crossover .

  • Uniform Crossover The crossover occurs on every gene unit with a determined possibility p.

  • Arithmetic Crossover The new gene comes out of the linear combination of two old genes, say $C=\lambda A+\mu B (-2\leq\lambda+\mu\leq2)$.

As for mutation, there are several options for binary/floating encoding chromosomes.

  • Simple Mutation. Each individual has a determined possibility to mutate on several randomized units.

  • Uniform Mutation. The chorosome will be totally and randomly replaced(this fits only in the primer stage).

  • Nonuniform Mutation A noise is added to the value, while Gaussian Mutation generate a value to replace on Gaussian distribution.

  • Boundary Mutation It offers a change giantly only in one dimension. For instance, the second gene locus of (1,2,9) can be shifted to the boundary value 0 or 9. This works well when the optimized solution is close to the boundary.

4. Gene Algorithm on 0-1 Knapsack Problem

The evaluation function: $f(x)=\sum v_ix_i, x_i\in${0,1}. The chorosome can be binary encoded to stand for a policy with n units.
