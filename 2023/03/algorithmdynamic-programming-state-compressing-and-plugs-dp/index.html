[Algorithm]Dynamic Programming: State Compressing And Plugs DP

March 18, 2023 in Algorithm, DP

State Compressing, which indicates a way to depict different states relatively succintly, are widely applied to two sorts of typical problems: the existence of a certain element in a set and 2D DP Problems concerning connectivity.

Generally, when an element is not in the set, we take 0 as a mark; conversely, 1 represents its existence in the set, which associates the binary form of an integer. Thereby, we take an integer N to represent the state of a set with log(2,N) elements at most.

Therefore, under the circumstance where we need to represent two states, it of necessity to bring in bitwise operations.

However, it also has a downside as the scale of the set rises up, the space and time complexity will rocket up with a exponential speed. Considering that “int” and “long long” represents 16 and 64 digits,such practice is not suggested when the set has more than around 12 elements in total.

Part Three: State Compression on Graphs

In most cases, State Compression is applied in the field of Compositions or Arrangements of Subsets where 0 and 1 stands for in-state or not-in-state. Sometimes we adopt ternary when two states are not enough.

Problem A: The Shortest Halmiton Circle

Problem You are given a directed graph whose edges has its weight. You are required to find one shortest Halmiton circle.

Solution If we dfs by force then it comes to be $O(n·n!)$ (n! paths in total and n times of addition) . With State Compression, we can lower n! to $2^n$ where the factorial scale is diminished to exponential scale.

We define f[S][j] where S is a subset of all points and its value means the shortest distance: the path begins from the source, traverses all the points in S and finishes at Point j. And the expression of the subset adopts states compression.

If we cast away j from S i.e. $S{j}$ then $f(S,j)=argmin(f(S-j,k)+d(j,k))$ . In this way we initialize $f(S)=0$ and let it run. The whole complexity is $O(n^22^n)$ where we enumate all subset states and update the distance by traversing $S$ for each point.

Problem B: A Weird Halmiton Circle

Problem Now we define a new type of circle, pass each point at least once and no more than twice. (The source is fixed) And now you are required to find the path whose cost is the least.

Solution It appears to be a complex version of Problem A. And this time we define $f[S][j]$ as well, but differently, S can have 3 states where 0,1,2 stands for having visited 0,1,2 times . But ternary is not natural for computer and therefore we choose quaternary.

The updating rule will be modified as $f(S,j)=argmin(f(S-j,k),f(S-2j,k))+d(k,j)$

Analysis: When facing plugging problems, we set 0 and 1 on each block. One block is 1 iff there is a horizontal one which strectches to the next column; it is 0 iff there is a vertical one or it is occupied by the horizontal one in the previous column.

We define dp(current row, state of current row) array as the number of ways. In view of its 1 by 2 scale, it is reasonable to compare the ith column and the i-1th. Therefore, we design the updating rule as we add dp(i,j) and dp(i-1,k) up iff there’s no conflict between the ith and i-1th, which ensure no intersections and blanks respectively. More exactly, state j AND state k can only be 0 and there cannot be odd consecutive zeros in j OR k.
