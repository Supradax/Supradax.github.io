[ML]Simulated Annealing

March 27, 2023 in ML

The Simulated Annealing Algorithm(SA) is a stochastic optimization algorithm inspired by the process of annealing in metallurgy. It starts with an initial solution and iteratively improves it by allowing it to jump to neightboring solutions with a certain probablity.

Principle

The crucial principle is to accept solutions worse than the current with a certain probablity, which is gradually decreased over time. This allows it to escape from local optima and explore the search space more effectively.

Parameters

  • Initial Temperature It should be set high enough to explore the search space effectively thus having a higher acceptance rate.
  • Cooling Scedule A common practice is to use a geometric cooling schedule that decreases the temperature by a constant factor (e.g. 0.95)
  • Step Size Learning rate.
  • Acceptance Probability It should be desgined to balance exploration and exploitation of the serach space. A common practice is to use the Boltzmann distribution with a temperature-dependent scaling factor
  • Stopping Criterion Based on the computational resources available. The upper bound of iteration times or a convergence point is mert.

Note.

  1. If it is expected to converge quickly, then we adopt a faster cooling schedule, a lower initial temperature, a smaller stepsize with a lower acceptance probablity while in the contrast if expected more effectively.
  2. Simulated annealing suits better in the context of a search within smaller but more complicated space while Differential Evolution fits a larger and smoother space; SA acts as a single-point algorithm while DE is based on a set of candidates.
  3. It is expected to attempt different parameters and initial positions and save the best position in time.
  4. Boltzmann Distribution where P(E) is the probablity of finding the system in a state with energy E. Z is the partition function which is the sum of the Boltzmann factors over all possible states of the system. k is the Boltzmann constant and T is the temperature. We can apply this to the SA where Ec and Es is the value of candidate and current vector. $$P(E)=\frac{1}{z} e^{-\frac{E}{kT}}$$ $$P(E_C,E_S,T)=min(1,e^{-\frac{E_C-E_S}{kT}})$$

More concrete steps are as below:

  1. Intialize the temperature t and initial solution x and design the cooling schedule and step size(learning rate)
  2. Repeat the following steps until the stopping criterion is met:
  • Select a neighbor Y by making a small change to X (step size)
  • Calculate the change in the objective function △f=f(Y)-f(X)
  • If △f<0 then we accept the answer and replace X with Y; otherwise accept with a probablity P(t) where t is the current temperature
  • Update the temperature t
  1. Return the current solution
