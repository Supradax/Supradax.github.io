[Algorithm]Dynamic Programming: Tree

March 20, 2023 in Algorithm, DP

Tree Dynamic Programming, which is a basic series of DP problems based on binary trees(or with more branches).

In DP problems, we always need to figure out how to transfrom one with large scale into a smaller and still maintain a complete structure . In segment DP problems, we transfrom a long one into a shorter one. This also rings true in tree – always look for a proper subtree and establish the state equation.

In segment problems, we have 2 orders to traverse the dp array(sequential and inverse); in tree problems, we can traverse from top to the bottom or from bottom to the top and use a dp array with an adjacent chart to store the result. In most cases, we have to combine backtracing with dp.

Problem The distance between two nodes is defined as the edges one has to walk through to another.

Analysis We can lay emphasis on a node. The longest path will either pass the node or not which enlighst us to set 2 state in our dp array. dp[u][] means the diameter of the subtree rooted on u while dp[u][0] implies u is not on the diameter and dp[u][1] does. In a binary tree, we have

  • dp[u][1]=dp[u->left][1]+dp[u->right][1]+1

  • dp[u][0]=max(dp[u->left][0],dp[u->left][1],dp[u->right][0],dp[u->right][1])

P.S. Actually there is such a theorem: For any node y, if we make it the root and begin to DFS then the farthest leaf must be one side of the diameter

[Proof] Suppose we begin from y and the real diameter is L(s,t) and the farthest node is z instead of t.

  • If y is on L(s,t): L(y,z)>L(y,t) i.e. L(x,z)>L(x,t) i.e. L(s,z)>L(s,t) i.e. L(s,t) is not the diameter
  • If y is not on L(s,t) and there is intersection between L(y,z) and L(s,t): L(y,z)>L(y,t) i.e. L(x,z)>L(x,t) i.e. L(s,z)>L(s,t)
  • If y is not on L(s,t) and there is no intersection between L(y,z) and L(s,t): L(y,z)>L(y,t) i.e. L(x,z)>L(x’,z)>L(x’,t) i.e. L(s,z)=L(s,x)+L(x,z)>L(s,x)+L(x,t)=L(s,t)

Thus we only need to choose y and find its farthest node A and take this node as the new root, then find the farthest node B. The length of AB is what we want.

Another interesting thing about the diameter is that: Despite a tree may have different diameter, but once the weight of the edges is positive then the midpoint will overlap [Proof] Suppose there is two midpoints x and x’ whose diameters are L(s,t) and L(s’,t’). Then L(s,t’)=L(s,x)+L(x,x’)+L(x’t)>L(s,x)+L(x,t)=L(s,t) (Distance Inequality)

Def The barycenter of a tree is a node which minimize the maximum of branches after deletion.
