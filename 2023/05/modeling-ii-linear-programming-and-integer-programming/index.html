[Modeling II] Linear Programming and Integer Programming

May 24, 2023

I 线性规划(LP)初步

  1. 问题构成：
  • 目标函数 $\max z = (L(x_i))$ (L是$x_i$的线性组合)
  • 约束条件 $L_1(x_i)≥k_1, L_2(x_i)>k_2,L_3(x_i)=k_3$
    其中目标函数求最小值可转换为求$\max -z$, 约束条件<k同理；且等式约束$L(x_i)=k$可转换为$L(x_i)≥k$或$L(x_i)≤k$

故一般地，可定义线性规划的标准型为目标函数、不等式约束、等式约束和上下界四部分构成 $$\min_x c^Tx s.t.$$ $$A_{ub}x≤b_{ub},A_{eq} x=b_{eq},x_i\in[lb,ub]$$ 2. 求解方法：单纯形法

易知在二维决策变量中，线性规划的可行域是一个一堆半平面的交集（不含等式约束，每多一个线性无关的等式约束使得维数下降1），在其中寻找到固定斜率的直线使得截距最大(小). 在三维情形中，可行域则为半空间的交，此时寻找固定法方向的平面使得z轴截距最大(小). 而且有以下性质：

  • 可行域R一定是凸集(凸集的交还是凸集)，但可能为空、封闭或无界
  • R非空时，最优解可能不存在(如$max x_1$ s.t. $x_1>0$)，最优解的个数可能是有限、无穷多个
  • 若存在有限最优解，则一定在R的某个顶点取到

对于形式上非线性规划的问题，但本质还是线性的，可通过恒等变形化为标准型：

  1. 绝对值的线性组合. $\min |x_1|+…+|x_n|$ s.t. $Ax\leq b$. 作分拆$|x_i|=\frac{|x_i|+x_i}{2}+\frac{|x_i|-x_i}{2}=u_i+v_i$，从而化归为$\min \sum{u_i+v_i}$ s.t. $u_i\geq0,v_i\geq0,A(u-v)\leq b$

  2. 最大(小)值的最小(大)值. $\min \max|q_ix_i|$ s.t. $Ax\leq b$，再定义新的决策变量和对应n个约束$y_1\geq q_ix_i,y_2\leq -q_ix_i$，分别计算最优解$\tilde{y_1},\tilde{y_2}$并取最大值$\max \tilde{y_1},\tilde{y_2}$

1.1 连续型调度问题（LP）

【例1】 某液体商品有m个产地，n个销地，各地产量、需求量为$p_i,d_i$，若将商品从产地i运到销地j的单位运价为$c_{ij}$，求：总运费最小的调运方案.

  Sol. 引入决策$x_{ij}$表示从i运到j的商品数，则目标函数为$\min \sum_{i,j=1}^{i,j=n}c_ijx_ij$，各地产量要求$a_k=\sum_{i=1}^n{x_{ki}}$，销售需求量要求$b_k=\sum_{i=1}^m{x_{ik}}$，取值范围为$x_{ij}\geq0$

  Notes.如果该商品按件计数(定义域在欧式空间中不连续)，则为整数规划问题

1.2 离散型调度问题（0-1规划）与匈牙利算法

【例2】 生产甲乙两种机床，利润为$y_1,y_2$. 生产甲、乙需用机器A、B、C分别加工(2,1,0),(1,1,1)小时，若每天三个机器可加工总时长为(10,8,7)小时，求：每天各生产多少台甲乙最优？

  设甲乙生产$x_1,x_2$台（选取合适决策变量），则目标函数为$\max z = x_1y_1+x_2y_2$，约束条件为$x_1,x_2≥0,2x_1+x_2≤10 x_1+x_2≤8,x_2≤7$

对于一般的LP问题，当决策变量维数为n时，定义满足L(x)=b的点集为超平面，若干个半空间的交集形成的区域为多胞形(空集是特殊的多胞形)，其中有界(封闭)的多胞形称多面体. 下面给出关于上文涉及的几何概念的数学定义：

  • 区域R是凸集 当且仅当 $\forall x,y\in R,k\in[0,1], kx+(1-k)y\in R$
  • x是多胞形R的极点(如平面多边形的顶点) 当且仅当 不存在$y_1,y_2\in R,k\in(0,1)$ 使得$ky_1+(1-k)y_2=x$

【例3】 分配n个人去做n项工作，每人仅做一项，定义花费矩阵{$c_{ij}$}($c_{ij}$表示第i个人做第j份工作的成本花费)，求：如何分配使得总花费最少?(即求一个置换$\sigma$使得$\min \sum{c_{i\sigma(i)}}$)    定义01决策变量$x_{ij}$，$x_{ij}=0$表示不让第i个人去做第j项工作，则目标函数为$\min \sum_{i,j=1}^{i,j=n}c_ijx_ij$，约束关系为$x_{ij}=0$或$1$.

匈牙利算法 原理为对一个成本矩阵C(必须是方阵)，对C初等行列变换不影响最优指派$\sigma$，可以通过机械的步骤求解：(定义独立零元素为方阵C中满足所在行列没有另一个0的元素)

  1. 每一行减去各行的最小元素，产生零元素
  2. 寻找独立零元素，从单个0的行开始，给0画圈，然后划掉所在列的
  4. 得到一个矩阵使得每个0所在行列没有0(对应一个置换矩阵)

若$c_{ij}=0/1$表示i和j之间不存在/存在边，则目标$\max \sum x_{ij}c_{ij}$表示在二分图$(G;V_1,V_2),|V_1|=|V_2|$中找到一个最大匹配. 可以看到，该问题是匈牙利算法的一个特殊情形，而该问题可用寻找增广路的方式求解.

  • 交替路：从未匹配点出发，依次经过非匹配、匹配…交替的路径
  • 增广路：从未匹配点出发，走交替路能到达另一个未匹配点(该路可通过交换匹配边和非匹配边，使得匹配边＋1)
