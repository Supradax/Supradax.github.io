[Algorithm]Dynamic Programming: Segment

March 19, 2023 in Algorithm, DP

Segment Dynamic Programming, which is a basic series of DP problems based on linear tables.

Update Records: 20/3/2023 Finish the Knapsack Session

To transfrom a problem into a DP problem, we need to ensure that:

  1. It has non-aftereffect property
  2. The problem can be transformed into a subproblem with smaller scale

In order to tackle a DP problem, we have to think over these problems:

  1. What do the value and the parameters of dp array mean?
  2. What is the state-transformation equation?
  3. How to initialize the dp array?
  4. Which order do you adopt to traverse the dp array?
  5. Print dp arrays if something goes wrong.

P.S. Sometimes due to the lack of states, we may not cope with the problem successfully. This time we may need to consider increasing a dimension in our state space.

Part One: 1D and 2D Problem(Primer)

Problem A: (1D)Climbing Stairs

Problem: You are climbing a staircase. It takes n steps to reach the top.

  1. Each time you can either climb 1 or 2 steps. In how many distinct ways can you climb to the top?
  2. You are given an integer array cost where cost[i] is the cost of ith step on a staircase. Once you pay the cost, you can either climb one or two steps. You can start from either stair 0 or stair 1.

Analysis:

  1. Take dp(i) as the numbers of ways you can reach stair i, then dp(i)=dp(i-1)+dp(i-2), which is supposed to be initialized as dp(0)=dp(1)=1.(If set to be 0,dp[] constantly equals 0)
  2. Take dp(i) as the least cost it requires to reach stair i, then dp(i)=min(dp(i-1)+cost[i-1],dp(i-2)+cost[i-2]), which is supposed to be initialized as dp(0)=dp(1)=0

Problem B: (2D)Unique Paths

Problem: There is an m x n grid.

  1. What you need to do is to figure out the account of reaching the right-bottom corner from the left-top corner without turning back(You can only go rightwards or downwards).
  2. Based on question 1, an obstacle and space are marked as 1 or 0 respectively in the grid. A path that you take cannot include any square that is an obstacle. The goal keeps the same.

Analysis:

  1. Take dp(i,j) as the number of methods you can reach (i,j), then dp(i,j)=dp(i-1,j)+dp(i,j-1), which is supposed to be initialized as dp(0,0)=dp(0,j)=dp(i,0)=1(0<i<m,0<j<n) Otherwise the entire dp array will be zero.
  2. This time we need to add an extra judgment to ensure its validity. dp(i,j) can be updated iff there is no obstacle on (i,j),(i-1,j) and (i,j-1). To initialize dp, still we need to set the values on the 0th row and column. The value will be 1 until it meets obstacle for the first time which makes the value behind 0.

Part Two: Knapsack Problem and Scrolling Arrays

§1 0/1 Knapsack Problem

Problem You are given a series objects with labels of their weight w[i] and value v[i]. You need to maxmize the value of the objects you take in a limit of weightk.

Analysis For each object you can simply choose to take or not which is quite similar to State Compression. However, we can finish this task more efficiently and handle a much larger scale than State Compression.

We define dp(i,j) as the maximum value you can gain where you collect things from the first ith objects with volumn j. For each object, you can choose to take it or not thus making dp(i,j)=max(dp(i-1,j)+0, dp(i,j-w[i])+v[i])

Now comes to the traverse order. The expression denotes that state(i,j) origins from state(i-1,j) and state(i,j-w[i]) making us to traverse from the left-upper corner to the right-bottom. (We have to ensure the origins updated before (i,j) has)

Question Whether can we shift the two loops? Although it is valid to take these two orders, however, the result may vary. If the volume loop lies in the exterior, then it will traverse all objects first. But anyway, it won’t take one stuff twice which means it is fine.

We observe the equation and discover that the update in object-dimension is only up to the previous state i-1 thus enabling us to compress the state space and that is the spirit of “Scrolling Arrays”

But this time some questions bubbled up, we have to consider the following problems:

  1. Can we shift two loops safely?
  2. Can we still adopt such traverse order?

For the first one, the interior loop will keep on filling one blank and then turn to the next one. Such behavior is exactly possible to take one stuff twice.

For the second, take w[0]=1 as an example, when j is 1 it chooses to take stuff 0 and such thing occurs again when j gets to 2. Therefore, it transfroms into a mutiple choice knapsack problem which does not match our intention. Therefore, we need to employ the reverse order instead of the forward order.

Problem A:Partition Equal Subset Sum

Problem Given an integer array nums, return true if you can partition the array into two subsets such that the sum of the elements in both subsets is equal or false otherwise. Sample

Input: nums = [1,5,11,5]
Output: true
Explanation: The array can be partitioned as [1, 5, 5] and [11].

Input: nums = [1,2,3,5]
Output: false
Explanation: The array cannot be partitioned into equal sum subsets.

Analysis In this scene, we can take a number once which associates with 0/1 knapsack problem. In view of convenience, we may not let weight and value equivalent as well. We can easily get the target sum which is half sum of the array k. Therefore, now we are going to set the target volume is k. Due to the equivalenence of weight and value, the max value it can get with volume k is no greater than k. The answer is true iff maximum value equals k.

Problem B: Last Stone Weight

Problem: You are given an array of integers stones where stones[i] is the weight of the ith stone. On each turn, we choose any two stones and smash them together. Suppose the stones have weights x and y with x <= y. The result of this smash is:

If x = y, both stones are destroyed, and if x != y, the stone of weight x is destroyed, and the stone of weight y has new weight y - x.At the end of the game, there is at most one stone left.

Return the smallest possible weight of the left stone. If there are no stones left, return 0.

Analysis: Intuitively, a greedy idea emerges up – to take the pair of stone with most similar weight which shift Problem B into Problem A But this time we don’t need to know whether dp[half sum] equals half sum; instead the answer comes from the D-value = sum-dp[sum/2]×2 .

Problem C:Target Sum

Problem: You want to build an expression out of nums by adding one of the symbols ‘+’ and ‘-’ before each integer in nums and then concatenate all the integers. Return the number of different expressions that you can build, which evaluates to target. Sample:

Input: nums = [1,1,1,1,1], target = 3
Output: 5
Explanation: There are 5 ways to assign symbols to make the sum of nums be target 3.

Analysis: This problem no longer asks us to return the maximum but instead the number of ways to fill out the bag. We rewrite the definition of dp array.

dp(i,j) is defined as the number of ways to fill out the bag from the first i objects with volume j and dp(i,j) = dp(i-1,j-w[i])+dp(i-1,j)
Likewise, we use the trick called scroll array then it becomes: dp(j)+=dp(j-w[i]). Now that it is 0-1 problem we should take the reverse order

For this problem, if the sum of numbers with positive sign is x, then the sum of those with negative sign is sum-x. Target=x-(sum-x) i.e. x=(target+sum)/2 Now we want to know how many ways are there to fill out a bag whose volume is x.

Problem D:Ones and Zeros

Problem: You are given an array of binary strings strs and two integers m and n. Return the size of the largest subset of strs such that there are at most m 0’s and n 1’s in the subset. A set x is a subset of a set y if all elements of x are also elements of y.

Input: strs = ["10","0001","111001","1","0"], m = 5, n = 3
Output: 4
Explanation: The largest subset is str0,1,3,4 making the answer 5.

Analysis: In this scenario, there are two kinds of weight– 0 and 1 Thus we first transform each string to the relative weight, say w0[]={1,3,2,0,1},w1[]={1,1,4,1,0} (take the sample as an example) while the value of each object is one (only contribute one element to the subset)

To sum up, algorithms to do with knapsack problems always have two target: whether can you fill out the bag(Problem A), maximize the value(Problem B) and count the ways to get a certain value.(Problem C) In the meanwhile, it may have more than one type of restriction(Problem D) or value

Part Three: Stock Problem and House Robber Problem

§1 Best Time to Buy and Sell Stock

Problem You want to maximize your profit by choosing a day to buy one stock and choosing a different day in the future to sell that stock. Requirement

  1. You can only purchase and sell once;
  2. On each day, you may decide to buy and/or sell the stock. You can only hold at most one share of the stock at any time. However, you can buy it then immediately sell it on the same day.
  3. You may complete at most two transactions.
  4. You may complete at most k transactions(k is given).
  5. After you sell your stock, you cannot buy stock on the next day (i.e. cooldown one day).
  6. You may complete as many transactions as you like, but you need to pay the transaction fee for each transaction. (The fee is given)
Input: [1,2,3,4]	everyday stock price
Output: 0	the most money you can gain

Analysis

  1. Pinning on the target we design the dp array as the most money and notice that whether or not you have sold the stock differs. Our state space is supposed to have at least 2 dimension , one for time and the other for the ownership of stock. Thus it turns to be dp(the ith day, state): 0 for unhold, 1 for hold, 2 for having sold. Updating rule: dp[i,0]=0, dp[i,1]=max(dp[i,0]-price[i],dp[i-1,1]), dp[i,2]=max(dp[i-1,2],dp[i-1,1]+price[i]) What we want is max(dp[n,2],dp[n,0])

Just so you know, such statespace can be cut to only two. We define 0 as unhold and 1 as hold. Updating rule would be dp[i,0]=max(dp[i-1,0],dp[i-1,1]+price[i]), dp[i,1]=max(dp[i-1,1],-price[i]) But this time, we have to consider the initialization carefully. If still all zero, the entire dp array will be all zero. Now let’s trace back to the meaning and find that on Day Zero, dp[i,0]=0, dp[i,1]=-price[0]. Notice that Day i is only up to the Day i-1. There we have:

  1. Now that we can sell and buy for many times, the original state before purchassing doesn’t have to be 0. Therefore, there is one thing to change: dp[i,1]=max(dp[i-1,1],dp[i,0]-price[i])

  2. Based on question 1, we have the intention to enlarge the statespace where 0 is for purchasing 1st stock, 1 for selling it out, 2 for purchasing 2nd and 3 for selling out again. Then we have: dp[i,0]=-price[i]; dp[i,1]=max(dp[i-1,1],dp[i-1,0]+price[i]),dp[i,2]=max(dp[i-1,2],dp[i-1,1]-price[i]),dp[i,3]=max(dp[i-1,3],dp[i-1,2]+price[i]) And the outcome will be max(0,dp[n,1],dp[n,3]) (We don’t really need to trade twice). The initialization will be dp[0]=dp[2]=-price[i], dp[1]=dp[3]=0

  3. It is identical to question 3. We enlarge the statespace to 2k-1 where 0,2,…,2n-2 means purchasing kth stock (initialized as -price[0]) and 1,…,2n-1 for selling it out (initialized as 0). And compute the equations: dp[i,2k]=max(dp[i-1,2k],dp[i-1,2k-1]-price[i]), dp[i,2k+1]=max(dp[i-1,2k],dp[i-1,2k]+price[i]). And the outcome will be max(n,2k-1)
