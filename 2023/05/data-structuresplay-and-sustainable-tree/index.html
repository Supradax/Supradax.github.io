[Data Structure]Splay and Sustainable Tree

May 17, 2023 in Data Structure

Splay Tree, a masterpiece by Tarjan, is a handy self-balanced BST.

常见的自平衡BST数据结构有伸展树(Splay Tree)、替罪羊树(Scapegoat Tree)、Treap、AVL树和红黑树等.

I. 普通Splay树

满足功能：

  • 插入和删除一个数(重复的数存在相同结点上)、
  • 查询某个数的前驱后继
  • 查询某个数的排名或某个排名对应的数

结点结构：

1.1 左右旋转与维持平衡性

和AVL不同，Splay树无需区分LL、RR、LR和RL四种关系，但仍有左右旋操作并且分直线和折线型进行旋转. 且任何操作都需将目标结点旋转至根结点的位置以维持平衡.

旋转： 涉及三条边的修正，对右旋而言，为z-y, x-y和x-b. 依次执行即可. 可用异或将左旋和右旋合并.

左旋与右旋示意图

伸展(Splay): 访问一个节点x，并把x旋转到根节点k. 其中若x的父节点y是根(y=k)，则做单旋；否则若为直线型(LL,RR)则双旋(尽管没有本质差异)成直线，若为折线型(LR, RL)，则旋转成V型.

Splaying Process

1.2 查找前驱后继、排名查询

  • find(): 找到v所在节点并转至根；由于find()已经将该节点转至根，故查询某数排名时只需看左子树的大小

  • get_suc(), get_prev(): 查询前驱、后继并返回编号，可先调用find(v)，若该节点查找不到，则返回上来的可能是其前驱或后继，则需先判断，若小于v则是前驱，反之则需找右孩子的最左叶子；后驱同理. 若等于v，则直接去叶子找. 后二者可合并.

  • del(): 若cnt不等于1，则查找到后cnt–；否则考虑把x的前驱转到根，后继转到前驱的下面(右孩子)，由夹逼可知，x一定是根节点的右孩子的左叶子，删除即可.(需要插入±∞两个哨兵，好处是删除最值时仍能找到前驱后继，无需特判)

1.3 插入与排名查询

  • get_val(int k):查询排名为k的数值. 若左子树小于k但左子树加上根节点cnt大于等于k，说明根为所求；否则若加上还小于，则在右子树中求得；再不然则在左子树中.

  • insert(int x): 利用BST性质，若找到则cnt++，否则单独加叶子节点再splay.

Notes. 一般应用时只需写pushup, rotate, splay和insert四个函数.

III. 可持久化线段树

3.1 可持续化线段树

在一些场景下，我们需在维护一个线段树的同时，保存其历史信息. 由于每次维护线段树只需修改一部分，这些修改部分新增节点存储，而未修改的分支则指向上一个版本的树的结点 由于未知空间大小且新增结点位置不是简单的2倍或2倍+1的关系，故需动态开点. 其中每开一个版本对应一个根节点

接下来递归建立各个版本的线段树，x、y为前一、当前版本的节点指针，其中要修改的y为传引用，因为每个++idx是在新建节点，而传统静态线段树不会在更新新开节点.

在更新时，先拷贝原有节点，这样就获得原有的关系，再将需要更新的结点自己开结点接过去.

若要查询版本[1,r]上的第k小，则找到插入r对应的历史版本二分即可(即BST上的排名查询)，而要查询[l,r]一系列版本中的第k小(相当于区间内的第k小，一次插入对应一个元素)，则类比前缀和，即两个版本左子树的大小之差(x、y为第l-1、r个版本)

一般可持续化线段树只支持单点查询+单点修改，故而pushdown的优化是不必要的，因为每次修改至多消耗O(logn)

对于浮点数或较大的数，则离散化处理，最简单的为用vector排序. 然后利用其下标0,1,…作为大小，查询到第k小为i只需访问nums[i]. 或者也可以动态开点

3.2 可持久化数组

要求支持修改和访问某个版本某个位置的值，得到新的版本. 则可利用可持久化线段树，只有叶子结点存储数据值，内部结点不存信息，每次操作消耗O(logn).
