[Algorithm]DFS and BackTracing, BFS

March 24, 2023 in Algorithm, Simulation

Breadth First Search，广度优先搜索与Depth First Search深度优先搜索是两大基本搜索算法，其中DFS一般效率相对较高，采用栈模拟或递归实现，常与回溯算法结合；BFS由于需要用队列存储当前层节点，达到类似层序遍历的效果，消耗资源较多，但一般是最短路径或有时序限制的搜索问题比较简单的解法(使用DFS可能绕进一条极曲折且极长的路，更优的有如A星等启发式寻路算法)

下面以二叉树为例做相关引入，一般地，二叉树只需考虑递归逻辑的返回逻辑(如到达叶子节点等)、节点处理逻辑(即父节点与子结点的逻辑关系)、信息传递(可直接pack一个info结构体避免递归只能递归一个基本数据类型的问题)

对n叉树逻辑没有太多变化，只需将原有左右子节点变更为一个记录所有子节点的邻接表

1. DFS基本操作与二叉树

i. 二叉树的前序、中序和后序遍历：基本dfs操作

前、中、后序遍历，即中间节点的遍历顺序的位置。前序遍历(preorder)经过即访问；中序遍历先经过入栈，直到再次到栈顶时进行访问；后序遍历(postorder)则直到没有子结点未访问时再访问。对应递归逻辑只需调换dfs位置，

另外也可使用栈模拟，其中前序遍历每遇到新节点即开始访问并压栈子结点；中序遍历则先入栈并遍历其子结点，直至该节点无左子结点时开始弹栈访问；后序遍历由于直接实现较难，采用技巧，调换左右访问次序并倒序输出，将中右左的前序遍历转置为左右中

ii. 判断对称树：双节点信息

Analysis 对root只有一个节点或为空进行特判，传入root的两个节点**同步反向(即同时但左右颠倒)**进行dfs返回即可

iii. 计算树的最大或最小深度：提前一层退出递归

Analysis 最大深度即所有叶子到根节点的最大值，最小深度同理. 其中最小深度不同之处不能等到该节点为空再返回0，否则对于形如链表的树会得到最小深度恒为1. 一般地，等到节点为空再作判断往往会丢失一些信息

变式 判断平衡二叉树(平衡指左右子树高相差不超过1)

iv. 寻找路径总和：剪枝与回溯初步

Problem 在二叉树中找到一条路径使该路径上所有结点权值之和为目标值

Analysis

  • 将统计节点数/高度的+1变为加上节点权值
  • 若给定权值均为正数，则可至少在2个地方剪枝，即当目前路径总和已经大于目标值时，可直接退出；已经找到目标路径时可返回
  • 在离开当前节点时，当前节点的权值必须减去(恢复原状，即递归完成后进行回溯处理)；若不使用该方法，则可通过递归参数传递让系统自己释放消除影响

v. 左叶子之和：传递两层信息

Problem 左叶子，即二叉树中属于父节点左侧的叶子。目标是返回所有左叶子的权值之和.

Analysis 必须通过父节点得知该节点是否位于左侧，类似最小深度中不能到达空节点再作返回判断，否则会丢失一些信息

vi. Misery Story: 反转二叉树

2. 回溯算法

回溯算法本身是dfs暴搜的一种(n重for循环)，只是侧重强调回溯的操作。一般适用于排列组合问题、切割问题、棋盘问题等，需要配合适当的剪枝和记忆化搜索降低时间成本

一般地，需要设计好以下问题：递归出口、传递信息、回溯过程 下面看典型题：

Problem A: 排列组合问题

Problem 给定n，输出1~n共n个数的全排列、所有组合(升序)

对于组合问题，差别在于for循环的开始和结束位置。而for的设计使得代码无需in[]标记，因为不可能出现重复元素进入

变式 组合总和类问题

  1. 从给定数组中取k个索引对应的数使其和恰为目标和. 同理将已选索引的上确界传入，进行剪枝并确定好for开始位置避免重复，回溯即一加一减
  2. 在(1)的基础下，不限制k的个数且已知正数数组中无重复元素. 同理确定for的开始位置，回溯一加一减，并由于为正数列，从而当sum>target时可进行剪枝
  3. 在(2)的基础下，数组有重复元素但输出的组合不能重复. 可以考虑用unordered_set；通过多重背包DP求解；或先进行排序，用pair构建键值对<值，数量>，每层dfs设置双重for循环，一个遍历值，一个遍历该值对应的选取数量

Problem C: 序列问题

Description 在给定数组中找到所有长度大于1的单调不减子序列 Analysis 类似组合问题，逐个dfs回溯即可

3. BFS 广度优先搜索基本操作

引例：二叉树的层序遍历

将初始节点加入队列，当队列非空时，记录每轮初始队列的大小(否则不知道该层的元素个数)，遍历当前层的各个节点同时，将其子结点加入队列，直至队列为空.

对于树以外可能存在回路的节点，需要加入vis[]数组标记是否访问或者使用flooding方法将该节点设置为不可访问

Problem A 迷宫问题：最短路问题

Description 给定一个m×n大小的0-1矩阵，1表示可达，0表示不可达，求从(0,0)到(m,n)的最短路径(不可达则输出-1)

Analysis 最短路问题使用BFS，每一层代表走一步，最先探测到终点的一定是步数最少的

Problem B 海岛问题

Description给定一个0-1矩阵，0表示水域，1表示陆地，求：(假定边界均为水域) Part 1 岛屿的数目、最大面积、最大周长、封闭岛屿的数目(不与边界相连)

Solution

  1. 岛屿数目可用并查集合并；BFS或DFS进行Flooding统计(或使用vis[]标注已访问的)，遍历整个图;

  2. 遍历每个岛屿并结合Flooding即可

  3. 遍历到陆地时，检测4个方向为水域的个数n，cnt+=n–

  4. 使用flooding方法先把边界进行不统计检索，淹掉不合题意的岛屿后再转换为第一问

Analysis 对该类问题，一般可以考虑DFS, BFS与并查集，末尾给出两套模板

Part 2

  1. 若同时进行“填海”操作，即令(i,j)=1，返回每次填海后的岛屿数目；
  2. 若能人为令一格从0变为1，求此操作后最大的岛屿面积；
  3. 若在一天内能进行一次灌水(从1变为0)，求使图不连通的最少天数（连通意味着只有一个岛屿）

Solution

  1. 并查集进行合并后，每次填海检测周围是否存在岛屿：若有且只有1个，则将其设为其编号；若有编号不同的，则用并查集合并并更新数目；若无，则单独编号
  2. 只有一格，故可遍历所有水域。将不同岛屿进行编号，若四个方向上有岛屿，则将所有不同号岛屿面积相加，再加1，维护该最大值
  3. 分类讨论：显然只有{0,1,2}三种可能，且0可通过判断原始连通性得知 (i)枚举所有陆地再判断连通性，O(m²n²) (ii)Tarjan算法：方格为节点，相邻引出边，则1意味着对应点为割点，tarjan求出其存在性，O(mn)

Part 3

  1. 若岛屿只能通过平移后重合，则认为两个岛屿全等，求：不同形状的岛屿数目
  2. 若岛屿可通过平移、镜像(左右/上下)、旋转重合，则认为全等，求：不同形状的岛屿数目

Solution

  1. 由于从上到下、从左到右遍历，从而DFS过程中遵循相同的遍历顺序（具体差异和方向数组有关），但两个形状相同的岛屿遍历次序一定相同，对应生成字符串比较放入unordered_set去重
