[Algorithm]Sorting Algorithm and the Applications

March 21, 2023 in Algorithm, Sort

Def We claim an algorithm stable iff for any a[i]=a[j] i<j in the sorted array still satisfies i<j. Def We claim an algorithm in place iff there’s no need to create a space more than O(1) to finish the task.

Section One. O(n²) level

1. Bubble Sort

One of the classic sorting algorithm: Swap two elements if they breaks the order. In the worst case for instance, you need to sort the reversed array into the sequential one which means you need to swap at most n times for the first element. Therefore the algorithm is valid and the time complexity is O(n²).

There exists many details waiting to optimize for us. For instance, if the ith traverse doesn’t execute any swaps, then we can simply break out.

Now look at the interior loop and we find that for the array 1,3,2,6,7. The 3rd and 4th number are already the greatest and they won’t be swapped in any cases then we can set the upper bound of interior loop to 2. There we have:

Now that we complete the optimization in the rear, why not do it again in the front?

And the final version is Cocktail Sort or Dual Bubble Sort

It is easy to know if there are two identical element, then bubble sort won’t be carried out thus making it stable .

The updating version with the same idea swapping is quick sort.

2. Selection Sort

In each traversal, we pick up the least number and swap which needs n times calculation. With n numbers in total, this algorithm takes n² times.

Optimization We may use another variable to record the global maximum and update the array from both sides which is similar to cocktail sort.

For selection sort, it is possible to change the relative order(4(a) 4(b) 1 -> 1 4(b) 4(a)) thus making it unstable. The optimized version with the same idea selection is Heap Sort but still it is unstable.

3. Insert Sort and Shell Sort

For insert sort, we plug in an element into an ordered sequence . Finding the proper position for each element takes O(n) thus making it O(n²).

For insert sort, its stablity depends on the way you insert the element. Generally we regard it as stable.

An idea developed by Shell is not to insert the element one by one; instead, pack them up first and minimize the scale step by step. For instance, if we have 256 numbers to sort, then it can be 128 numbers in a pack for the first time and then 64,32,16 on and on and on. Then apply insert sort respectively. However, the complexity is unknown according to the gap you choose. But at least it is believed that it will be better than O(n²) but no better than O(nlogn). And the last gap has to be 1 to guarantee the correctness(Because this moment the array will be almost ordered thus making it O(1) in most cases).

By setting parameters o Diminishing Increment Sort we may get different efficiency despite the average of time complexity turns out to be n to (the power of)1.3

Section Two. O(nlogn) level

1. Merge Sort

Each time we divide (break an array into two until each one become isolated) and conquer(merge two ordered arrays and become an integration in the long run).

As for the optimization, we can use the stack to simulate the recurssion and compare the last in the left aand the first in the rightb when merging – if a≤b then they are already ordered.

Problems to do with Idea of Merge Sort often are about some certain relation we can get during merging

Problem We are wondering how many reverse order pairs are there in a given array. Analysis We will find a lot when putting in numbers on right the side. This practice means those left on the left(in the front) are all greater than this number put in.

2. Quick Sort

Quick Sort also applies the idea of making choices. The entire process is that we choose one element from the array as the criteria, those greater will be on the right side while the less on the left and the criteria will fall on the only one blank in the end

3. Stack Sort

In C++, we call the container priority_queue. According to the characters of heap, each time we find the greatest takes O(1) and updating takes O(logn) thus making it O(nlogn) in total.

IV. O(n) level

1. Bucket Sort, Counting Sort

Bucket sort and counting sort is exchange time with space and counting sort is actually a special circumstances of bucket sort. Therefore, let’s talk about counting sort first.

Suppose you have a series of number which ranges from 1 to 100 and we want to print out the sequential series. Thus we create an array with the space of 101 ints and traverse the array and put them in.

Then what we need is simply to traverse the bucket and we know the times of each number. You can find that only when the range of number is tolerable can we adopt such algorithm.

Back to bucket sort, we divide a large scale of number into some intervals, say, cut [1,1e5] into [1,1000],[1001,…] … [9901,1e5] and apply other sorting algorithms. When the bucket amount m equals that of elements, it degenerates into counting sort. The time complexity would be O(mf(n)), f(n) is up to the algorithm you adopt.

2. Radix Sort

For those have the features like each parameters with different piority, we can take radix sort into account. For instance, each 9 in 9999 definitely has different weights. And we notice that the compositions of each place is little enough that we can prepare some buckets(0-9 in total).

Then we put the numbers in accordance to the last figure of each number and get them out from 0 to 9 and then comes to the last second figure … until each non-zero figure has been traversed.

To make things easier, we can convert numbers to string and get them reversed.

The space complexity is O(k) and the time complexity is O(mn) where k is the sum of composition amount of each parameters (bucket) and m is the parameters ammount.

V. Summary

It’s easy to know that algorithms are simply making selections between space and time.
