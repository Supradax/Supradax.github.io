[Algorithm]Computational Geometry I: Basic Vector Operations

April 8, 2023 in Algorithm

Vector and Matrix are widely used in the region of computational geometry with which we can easily express the relative positions among points, lines and planes.

1 Vector

The code below offers an implement of Point,Line and Vector Class which support the dot product and cross product operations in a 2D plane.

1.1 The position of points

To know whether a point falls on the left or right side of a line ,cross product matters which is determined by the sign. If three points lands on the same line, then $\vec{AB}×\vec{BC}=0$

Take the sample problems beneath into account. Problem 1 The location of a series $A_1,A_2,..B_1,B_2…$ ascending points on $y=0,y=1$ is given($A_iB_i$ doesn’t intersect) and there is an isolated point $P(x,y)$. You are required to tell the point is located between Segment i and Segment i+1 which within O(logn). If P lands on a certain segment, then let the two value equal.

Solution We use binary search because each time the correct answer always lands on one certain half part. If the cross product is positive, then we search in the left side.

Problem 2 A center point P and point set A is given. You are required to search as many points as possible within a semicircle whose radius is r.

Solution To avoid the compact of R which is impossible for modern computers to solve, we use for reference the core idea of scanning line – instead of thinking over the entire line, we focus on the meaningful two endpoints. Likewise, we can see that there can be at least a point on the semicirle edge in the most optimized case(even though it doesn’t always have to). Therefore we firstly pick out all points within r and then rotate based on the given points.

1.2 The relative position between lines

With the help of vectors, we can distinguish the corresponding state of two lines or segments.

  • Figure (a). To judge if a point lies in the counterclockwise side of a line iff $\vec{AB}×\vec{AC}>0$.
  • Figure (b)&(c). To judge **if a segment intersects a line ** iff $|\vec{AB}×\vec{AC}||\vec{AB}×\vec{AD}|<0$ which indicates that the two endpoints C and D locate in different side of Line AB.
  • To judge if two segments have no intersection iff ** $ |\vec{AB}×\vec{AC}||\vec{AB}×\vec{AD}|>0$ OR |\vec{CD}×\vec{CA}||\vec{CD}×\vec{CB}|>0 ** which gains insight from the 2nd case – such criterion depicts two possible states: \vec{AB}//\vec{CD} and not parrelled(they will intersect if you prolong them and won’t when they stay on the same side of the other).

Notes. Scenes in Figure (b)&(c) might degenerate when the intersection is exactly Point A where the cross product becomes zero. Fortunately, it is contained in the design.

Problem 1 Piles of sticks. A lot of straight sticks are given in the form of the endpoints location $(x_1,y_1),(x_2,y_2)$ which are thrown on the floor in order. If two stciks intersect, then we group them into one pile. You are required to tell how many piles there are in total against the floor.

Solution The classification request assosiates with the DSU and for the newly-appended segment we apply cross product with each enumerated segment.

Problem 2 Common Projection. To find a line such that the projection of all segments onwards intersect.

Solution Likewise, we need to think over those crucial points. If we can discover a line l which have common intersections among all then the perpendicular line is the one we need. If we rotate l a little bit and the property remains, then the critical case is that it gets across an endpoint thus making us enumerate the lines determined by each two endpoints. The time complexity is O(n²).

Problem 3 Line of Sight A finite segment is given(Blue) and the endpoints and the height of several obstacles(Green) are offered($[A_i,B_i,C_i]$). Now you are required to figure out the longest range you can see on the blue segment.

Solution Calculate the intersections of the blue line and the vision line and sort them up. Apply scanning line or greedy idea to merge the intervals to delete or work it out directly with scanning line.
