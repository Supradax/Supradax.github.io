[Algorithm]Computational Geometry I: Basic Vector Operations and Convex Hull

April 8, 2023 in Algorithm

Vector and Matrix are widely used in the region of computational geometry with which we can easily express the relative positions among points, lines and planes.

1 Vector

The code below offers an implement of Point,Line and Vector Class which support the dot product and cross product operations in a 2D plane.

1.1 The position of points

To know whether a point falls on the left or right side of a line ,cross product matters which is determined by the sign. If three points lands on the same line, then $\vec{AB}×\vec{BC}=0$

Take the sample problems beneath into account. Problem 1 The location of a series $A_1,A_2,..B_1,B_2…$ ascending points on $y=0,y=1$ is given($A_iB_i$ doesn’t intersect) and there is an isolated point $P(x,y)$. You are required to tell the point is located between Segment i and Segment i+1 which within O(logn). If P lands on a certain segment, then let the two value equal.

Solution We use binary search because each time the correct answer always lands on one certain half part. If the cross product is positive, then we search in the left side.

Problem 2 A center point P and point set A is given. You are required to search as many points as possible within a semicircle whose radius is r.

Solution To avoid the compact of R which is impossible for modern computers to solve, we use for reference the core idea of scanning line – instead of thinking over the entire line, we focus on the meaningful two endpoints. Likewise, we can see that there can be at least a point on the semicirle edge in the most optimized case(even though it doesn’t always have to). Therefore we firstly pick out all points within r and then rotate based on the given points.

1.2 The relative position between lines

With the help of vectors, we can distinguish the corresponding state of two lines or segments.

  • Figure (a). To judge if a point lies in the counterclockwise side of a line iff $\vec{AB}×\vec{AC}>0$.
  • Figure (b)&(c). To judge if a segment intersects a line iff $|\vec{AB}×\vec{AC}||\vec{AB}×\vec{AD}|<0$ which indicates that the two endpoints C and D locate in different side of Line AB.
  • To judge if two segments have no intersection iff $|\vec{AB}×\vec{AC}||\vec{AB}×\vec{AD}|>0$ OR $|\vec{CD}×\vec{CA}||\vec{CD}×\vec{CB}|>0$ which gains insight from the 2nd case – such criterion depicts two possible states: $\vec{AB}∥\vec{CD}$ and not parrelled(they will intersect if you prolong them and won’t when they stay on the same side of the other).

Notes. Scenes in Figure (b)&(c) might degenerate when the intersection is exactly Point A where the cross product becomes zero. Fortunately, it is contained in the design.

Problem 1 Piles of sticks. A lot of straight sticks are given in the form of the endpoints location $(x_1,y_1),(x_2,y_2)$ which are thrown on the floor in order. If two stciks intersect, then we group them into one pile. You are required to tell how many piles there are in total against the floor.

Solution The classification request assosiates with the DSU and for the newly-appended segment we apply cross product with each enumerated segment.

Problem 2 Common Projection. To find a line such that the projection of all segments onwards intersect.

Solution Likewise, we need to think over those crucial points. If we can discover a line l which have common intersections among all then the perpendicular line is the one we need. If we rotate l a little bit and the property remains, then the critical case is that it gets across an endpoint thus making us enumerate the lines determined by each two endpoints. The time complexity is O(n²).

Problem 3 Line of Sight A finite segment is given(Blue) and the endpoints and the height of several obstacles(Green) are offered($[A_i,B_i,C_i]$). Now you are required to figure out the longest range you can see on the blue segment.

Solution Calculate the intersections of the blue line and the vision line and sort them up. Apply scanning line or greedy idea to merge the intervals to delete or work it out directly with scanning line.

1.3 Division:Euler Theorem

Some typical problems are like:

  • How many isolated regions are there when divided with n lines, n v-shaped lines or n circles?
  • What about dividing with n lines and m circles?

If we use mathmatical induction and apply greedy idea, then each time we put an additional line in, it is expected that the new line intersects with every line put in previously and the intersections are different. If it goes that way, we have n new intersections and n+1 new regions thus making it $a_n=a_{n-1}+n,a_0=1$ i.e. $a_n=\frac{n(n+1)}{2}+1$

Likewise, when it comes to be V-shaped polyline, each newly-built ray or segment can contribute to two extra regions. However, on account of the V-shaped, the amount decreases one at its vertex i.e. $a_n=a_{n-1}+2*n-1,a_0=1$

Now it comes to be circles. We also expect there appear more intersections. For a quadratic curve it can have at most two intersections among each. Therefore supposing there already exist n circles, then for the n+1th it can have 2n intersections at most which contribute to 2n new segments. i.e. $a_n=a_{n-1}+2(n-1),a_0=1$

It is also the same when it comes to n lines with m circles which requires us to use dp. A line can have two intersections with each circle thus $f(i,j)=f(i-1,j)+2j$ .Likewise, a circle can also have two intersections as well i.e. $f(i,j)=f(i,j-1)+2j$ And the initial condition is $f(0,0)=1$.

Problems can transform into a space in a higher dimension, say, how many regions are there when divided with n planes in a space? We can make an analogy with the case in a plane. We focused on the intersection point before and this time the intersection line is the candidate. And in the simplest case, such a line corresponds 4 regions in total. And thus we have $a_{n+1}=a_n+4n$ which make us suppose that in an m-D space, it should be $a_{n+1}=a_n+2^{m-1}n$.

Likewise, when it is divided by spheres, a new sphere will be divided by the n spheres whose amount is the same as that of maximum regions divided by n circles on a plane(Using Projection on a Plane to prove this). $b_{n+1}=b_n+2a_n$ (Those segment now become faces)

3 Convex Hull Theory

A polygon convex hull is defined as a polygan that for any inner point A,B, points on the segment AB are also inner points . With convex hull, we can deal with problems to do with some structures of the points set on their maximum or minimum states more easily.

3.1 Generate Convex Hull

The two typical algorithms are Graham-Scan and Andrew Algorithm .

  • Graham-Scan
  1. Find the lowest and leftmost point $P$(The vertical poisition is prior to the horizontal).
  2. With the pole $P$, sort up the points in the ascending order of the Polar Angle
  3. Traverse all the points with stack. If the new point sits on the right side of the line or simply sits on the line(determined by the two points on the top) on the stack top, then pop out the top element until this doesn’t occur.
    The Process of Graham-Scan
    The time complexity is O(nlogn) where the sorting process is in the majority.
  • Andrew It can be seen that a convex can be divided into an upper and lower one whose x-coordinates increase monotonically.
  1. Sort up the point set according to its x-coordinates
  2. The judging method is the same as Graham-Scan and then we obtain the lower part of the convex hull.
  3. Redo this again and skip those whose y-coordinates is lower than that of the half convex hull endpoints.

This algorithm avoid some float operations thus reaching a higher level on efficiency.

3.2 Rotating Calipers

Antipodes or Antipodal Points are a pair of points located in the two endpoints of a shape. For instance, two poles are antipodes on the sphere. Likewise ,we can expand this concept to edges such as V-V, V-E, E-E antipodes . And now we discuss about antipodes on the convex hull. And it can be proved that a N-polygan can have \frac{3n}{2} pairs of antipodes at most.

To find the antipodes, the corresponding method is Rotating Calipers which can diminish the complexity from O(n²) to O(n).

The spirit of the algorithm fully use the properties of convex shape(monotonically increase and decrease).

4.1 The Smallest Circle

Problem You are required to find a smallest circle to cover all the give points.

  • Lemma 1 The smallest circle is unique.

    Reductio Suppose there exist two circles and all points must be in the intersection. However, for any intersection of two circle, we can always construct a smaller one whose diameters is the two intersection points. It contradicts.

  • Lemma 2 For the circle $C_n$ which covers n points , when $P_{n+1}$ added, if $P\notin C_n$, then $P\in C_{n+1}$

    Reductio If not, then the green circle can maintain tangent against $C_n$ and shrink a little bit until it passes $P_n$.

The property revealed by lemma 2 shows that we can use induction which is called Incremental Method(增量法) . And Lemma 1 shows that whichever order we add the points, we can get the same result which means we can apply Randomization to decrease the probility of extreme data distributions – mix up the data randomly.

4.2 The Smallest Square

Problem The request remains the same but this time we use a square to cover.

Solution Square doesn’t have such property well enough like circles but at least we know if the edges are parrell to the axies, then we only need to search for 4 points s.t. $argmin(max(max(|\Delta x|),max(|\Delta y|)))$ .
