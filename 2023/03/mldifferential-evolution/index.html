[ML]Differential Evolution

March 26, 2023 in ML

Differential Evolution(DE) algorithm is a stochastic optimization algorithm that is insipred by the process of natural selection. The basic princple of DE is to evolve a population of candidate solutions(vectors in most cases) to find the optimal selection for the given problem
(usually to find the minimum or maximum of an expression).

Principle

DE starts with an initial population of candidate solutions, which are randomly generated vectors within the search space. At each iteration, a new population of candidate solutions is created by combining and mutating exisiting solutions from the previous population

The mutation step is performed by adding a scaled difference vector(we use a parameter called “scaling factor”) between two randomly selected vectors from the previous population to a third vector also randomly selected from the previous population. This generates a brand new vector but a little bit different from the original.

The crossover step is performed by comparing each element in the new candidate vector with the corresponding element in the original vector. If the element is smaller than a randomly generated crossover probability, the element is replaced with the corresponding element in the new candidate vector.

The resulting candidate solution is then evaluted using fitness function and we let the better solutions to leave to ensure the population size constant and the process is repeated until some certain criterias are met(the value comes to a convergence point or it is going to surpass the maximum iterations etc).

Parameters

  • Objective Function and Bounds The one you expect to optimize

  • Population Size(popsize). The number of candidate solutions(vectors) during each generation. It remains the same in the entire process.

  • Crossover Rate(CR) The probability that a given component of the trial vector is copied from the mutant vector

  • Scaling Factor(F) The constant used to control the amplification of the difference vector in the generation of trial vector

  • Termination Criterion It decides when to stop.

More concrete steps are as below:

  1. Initialize the population. Create N individuals randomly within the search space
  2. Evaluate the fitness Evaluate the fitness of each individual in the population based on the objective function to optimize
  3. Mutation Generate a trial vector by combining the vectors of 3 randomly selected individuals in the population The trial vector is created as follow. X=A+SF×(B-C) where A,B,C are three individuals in the population and X is the one generated.
  4. Crossover Perfrom crossover between the trial vector and the original target vector Y using a random binary mask to select which elements of trial vector to keep.
  5. Selection Compare the fitness of the trial vector X and the original target vector Y. If the trial vector performs better, then replace Y with X in the next generation; otherwise, simply keep Y and cast away X.
  6. Repeat Repeat step 2-5 until a termination condition is met.

During the loops, we keep X evolving until reach the one we expect. And we use the updating formula X=A+F(B-C)which is differential. And that is why it is called differential evolution.
